# TensorKiln v0.1.0-alpha.1

Verified graph front end and storage-planning core.

This is an experimental, source-only prerelease. It is not yet an optimized
tensor runtime: layout lowering, kernels, and optimized execution are not
available, and public APIs may change before stable v0.1.0.

## Shipped boundary

- checked static `f32` types and immutable, owner-safe graph construction;
- `Input`, `Constant`, `Add`, `MatMul`, and `Relu` shape inference and reference
  execution;
- deterministic dead-code elimination and exact structural canonicalization;
- composable provenance for graph-to-graph rewrites;
- deterministic 64-byte interval planning with an independent verifier;
- graph-derived storage lifetimes accepted only after independent reverse
  reconstruction agrees with the forward result.

The arena output is a non-executable storage projection. It retains dead
compute unless the caller explicitly runs dead-code elimination, leaves inputs
and constants external, and keeps graph outputs live through the final compute
step.

## Evidence

The tagged source passes:

- 170 dependency-free C++ tests;
- GCC 14 debug and release builds;
- Clang 18 debug and release builds;
- Clang 18 AddressSanitizer and UndefinedBehaviorSanitizer checks;
- an isolated Python-stdlib oracle for committed numerical fixtures.

The CI environment is Ubuntu 24.04. The project makes no compatibility claim
for untested operating systems or compiler versions.

## Verify from source

The release is distributed through GitHub's source archives. From the
repository root:

```bash
make -j2 test
make oracle
make -j2 example
```

GNU Make, Python 3, and a C++20 compiler are required. TensorKiln has no runtime
third-party dependency and does not provide an install target or packaged
binary in this prerelease.

## Not included

- target operations `Mul`, `Gelu`, `Softmax`, `LayerNorm`, `Reshape`, and
  `Transpose`;
- fusion, layout lowering, view or in-place alias proof, and scratch lowering;
- kernel selection, constant prepacking, an executable arena, or an optimized
  executor;
- benchmark or cache-performance claims;
- ONNX import, dynamic shapes, training, quantization, GPU, or distributed
  execution.

The complete stable target and explicit non-goals remain in the
[v0.1.0 charter](../charter.md).
